Definition of a polynomial tree object.

Parameters

  • splitting_criterion (str) â The type of splitting_criterion to use in the fit function. Options include model_aware which fits polynomials for each candidate split, model_agnostic which uses a standard deviation based model-agnostic split criterion [1], and loss_gradient which uses a gradient based splitting criterion similar to that in [2].

  • max_depth (int) â The maximum depth which the tree will grow to.

  • min_samples_leaf (int) â The minimum number of samples per leaf node.

  • order (int) â The order of the generated orthogonal polynomials.

  • basis (str) â The type of index set used for the basis. Options include: univariate, total-order, tensor-grid, sparse-grid and hyperbolic-basis

  • search (str) â The method of search to be used. Options are grid or exhaustive.

  • samples (int) â The interval between splits if grid search is chosen.

  • verbose (bool) â For debugging

  • all_data (bool) â Save data at all nodes (instead of only leaf nodes).

  • split_dims (list) â List of dimensions along which to make splits.

Sample constructor initialisations:

import numpy as np
from equadratures import *

tree = polytree.PolyTree()

X = np.loadtxt('inputs.txt')
y = np.loadtxt('outputs.txt')

tree.fit(X,y)
