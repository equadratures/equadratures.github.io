OVERVIEW

equadratures is an open-source model development platform. The models built in equadratures are explainable, robust, and do not require large cloud infrastructure to train. Additionally, equadratures' models can audit themselves as well as audit models built from other platforms - ushering a new era in machine intelligence.
equadratures started out as a code for efficient uncertainty quantification in engineering. It has saved countless hours in otherwise expensive Monte Carlo simulations by using a variety of polynomial chaos methods. Following the success of its polynomial chaos tools, equadratures has broadened in scope and is now used for model development, data analysis, and data visualisation.


METHODS

equadratures builds models using orthonormal polynomials over subspaces, graphs, and trees. The models are built in a manner to ensure that even with a small amount of data, explainable and robust models are always a few lines of code away. The chart below provides an overview of some of the key methods baked into the code.


The code and syntax are easy to pick up, with some commands as shown below. For more such examples, check out the code documentation and download the code! 

# Variable projection (Subspace-based polynomial generation)
import equadratures as eq

data = eq.datasets.load_eq_dataset('3Dfan_blades')
X  = data['X_c']
y = data['y1_c']

M = eq.Subspaces(method='variable-projection', sample_points=X, sample_outputs=y)
p = M.get_subspace_polynomial() # returns the subspace polynomial
# Polynomial chaos with relevance vector machines
import equadratures as eq

x1 = eq.Gaussian(mean=3.2, variance=1.5, order=3)
x2 = eq.Uniform(lower=-2., upper=13.5, order=4)
basis = Basis('total-order')
poly = Poly([x1, x2], basis, method='relevance-vector-machine', \
        sampling_args={'sample-points': X, sample-outputs':y})
poly.set_model()
poly.get_mean_and_variance() # returns the mean & variance
